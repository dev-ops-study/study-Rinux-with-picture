# 컴퓨터 시스템의 개요

## 컴퓨터 시스템 동작할 때 하드웨어에서 일어나는 일
1. 입력 장치 혹은 네트워크 어댑터를 통해 컴퓨터에 처리 요청이 들어감
2. 메모리에 있는 명령을 읽어 CPU에서 실행하고 그 결괏값을 다시 메모리의 다른 영역에 기록
3. 메모리의 데이터를 하드디스크나 SSD 등의 저장 장치에 기록 또는 네트워크를 통해 다른 컴퓨터에 전송하거나 디스플레이 등의 출력 장치를 통해 사용자에게 결과를 보여줌
4. 1번부터 반복 실행

## 프로그램의 종류
1. 애플리케이션: 사용자가 직접 실행하는 프로그램
2. 미들웨어: 여러 가지 애플리케이션이 공통으로 사용하는 처리를 묶어서 애플리케이션 실행 도움
3. OS: 하드웨어를 직접 조작하여 애플리케이션이나 미들웨어의 실행에 필요한 기능 제공

## 리눅스(OS)의 역할
- 외부 장치를 조작함. OS가 없으면 여러 개의 프로세스가 각각 디바이스 조작하는 코드 작성 필요.
- 없으면 상당히 귀찮아진다는 말

## 커널모드와 사용자모드
프로그래머의 버그나 해킹 목적으로 의도된 특정 프로세스가 디바이스 드라이버를 통해 디바이스에 접근함이라는 룰을 깨면 멀티 프로세스가 동시에 디바이스를 조작하려고 시도하는 상황 발생함. 

이러한 문제를 피하기 위해 CPU에 있는 기능 이용해 프로세스가 직접 하드웨어에 접근하는 것을 차단. 이를 커널모드와 사용자모드로 구분함. 

커널 모드에 있을 때만 디바이스 접근 가능. 

### 커널 모드
커널 모드에서 동작하는 OS의 핵심 부분이 되는 처리를 모아 담당하는 프로그램을 커널이라고 한다.
